# Hedge Tool

一个专为 Varitional 交易所 和 01.xyz 等平台设计的**快速对冲点击工具**。

通过预设两个屏幕坐标（A点：多头 / B点：空头），一键（或按 F9）快速顺序点击，实现对冲操作。  
支持可调节延时、多次执行、状态反馈，适合高频/手动对冲场景。

**当前版本特性**：
- 视觉标记（蓝色A点 + 红色B点）
- F9 热键一键触发（取代旧版循环监听）
- 可调点击延时（移动、按压、点间间隔）
- 支持单次/多次连续执行
- 实时状态提示
- Windows 高DPI 屏幕兼容

## 环境要求

- **操作系统**：Windows 10 / 11（64位推荐）
- **Python 版本**：Python 3.8 ~ 3.12
- **所需第三方库**：
  ```bash
  pip install pyautogui keyboard
  ```
  （tkinter 是 Python 内置库，无需额外安装）

## 安装步骤

1. 确保已安装 Python（推荐 3.10 或 3.11）
2. 打开命令提示符（CMD）或 PowerShell，执行：
   ```bash
   pip install pyautogui keyboard
   ```
3. 将脚本保存为 `hedge_tool.py`（或任意文件名，但建议保持英文/小写）
4. 双击运行，或在命令行输入：
   ```bash
   python hedge_tool.py
   ```

## 使用步骤（详细）

### 1. 启动程序
- 运行脚本后，会弹出一个**始终置顶**的小窗口（约 400×500 像素）。
- 窗口标题：**对沖神器 v4.8 - 序列執行版**

### 2. 设置坐标（最重要一步）

1. 打开你的交易平台（Varitional 或 01.xyz），调整到需要对冲的界面。
2. 把鼠标光标**移动到多头按钮**（或你要先点击的位置），保持不动。
3. 在工具窗口点击 **“設定 A 點 (多頭)”** 按钮。
4. 程序会显示“3秒後擷取座標...”，**3秒内不要移动鼠标**。
5. 3秒后，屏幕上会出现一个**蓝色圆圈标记**（A点），表示已记录。
6. 用同样方法设置 **B点（空头）**，完成后会出现**红色圆圈标记**。

> **小技巧**：  
> 如果标记位置不准，可以点击“清除所有標記”重来。  
> 标记是半透明、可穿透点击的，不会挡住按钮。

### 3. 调节延时参数（强烈推荐根据电脑/平台调整）

在界面中部有三个滑块：

- **移動延時**：鼠标移动到目标后等待多久再点击（默认 0.02 秒）
- **按壓延時**：按下鼠标左键后保持多久再松开（默认 0.05 秒，模拟真人）
- **點間間隔**：A点点击完成后，到B点前的等待时间（默认 0.02 秒）

**建议初始值**（可根据实际情况微调）：

| 场景               | 移動延時 | 按壓延時 | 點間間隔 |
|--------------------|----------|----------|----------|
| 高速电脑 + 响应快  | 0.01~0.02 | 0.03~0.05 | 0.01~0.03 |
| 普通电脑 / 平台延迟 | 0.03~0.05 | 0.06~0.08 | 0.05~0.10 |
| 非常保守（防丢单）  | 0.08~0.15 | 0.10~0.15 | 0.10~0.20 |

### 4. 设置执行次数

- 在“執行次數”输入框填写数字（默认 1）。
- 填 1 = 单次对冲
- 填 5 = 连续执行 5 次对冲（每次 A→B 完整序列）

> 多次执行时，程序会在每次序列间自动加 0.1 秒间隔，避免太快被平台风控。

### 5. 执行对冲

**两种方式任选其一**：

- **方式1**：直接点击窗口中的红色大按钮 **“⚡️ 點擊對沖 (F9) ⚡️”**
- **方式2**：按键盘上的 **F9** 键（推荐，操作最快）

执行过程中：
- 底部状态栏会显示“執行中... (X次)”
- 完成后显示“完成 (X次) - HH:MM:SS”
- 控制台（命令行窗口）会打印每次执行的坐标信息

### 6. 常见问题处理

| 问题                             | 可能原因                     | 解决方法                              |
|----------------------------------|------------------------------|---------------------------------------|
| 点击没反应                       | 延时太短 / 坐标不对          | 增大延时 + 重新设置 A/B 点            |
| 标记显示位置偏移                 | 高DPI 屏幕未正确处理         | 确保脚本最上方有 DPI 感知代码         |
| F9 无效                          | 键盘钩子被其他软件占用       | 改用鼠标点击按钮，或重启电脑          |
| 程序卡住 / 无响应                | 多次执行 + 延时过长          | 按 Ctrl+C 在命令行终止，或关闭窗口    |
| 平台检测到异常 / 风控            | 动作太机械                   | 增大所有延时 + 不要频繁使用           |
